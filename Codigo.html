CODIGO

Microsoft .NET

.NET es una plataforma de desarrollo de código abierto, gratuita y multiplataforma creada por Microsoft para crear muchos tipos diferentes de aplicaciones. En este proyecto, se ha utilizado el entorno de ejecución de .NET para garantizar la estabilidad y el rendimiento de la aplicación de escritorio.

Gracias a la integración con Visual Basic.NET, el software aprovecha las bibliotecas de clases de Windows Forms para ofrecer una interfaz de usuario intuitiva y una gestión de archivos robusta.

Gestor Avanzado de Archivos: Compresión y Conversión

Nuestra aplicación surge como una solución integral ante la necesidad de gestionar archivos de forma masiva y eficiente. El software permite a los usuarios no solo reducir el peso de sus documentos mediante algoritmos de comprensión, sino también adaptar sus archivos a diferentes entornos mediante el cambiador de extensiones.

1. Lógica del Cambiador de Extensiones

El módulo de conversión permite renombrar la extensión de uno o varios archivos simultáneamente. A continuación, se muestra el ejemplo de cómo el programa procesa el cambio de nombre en el sistema: 


' Ejemplo de lógica para cambiar extensión en VB.NET
Dim rutaOriginal As String = "C:\MisDocumentos\archivo.txt"
Dim nuevaRuta As String = IO.Path.ChangeExtension(rutaOriginal, ".pdf")

' Realiza el cambio físico en el disco
IO.File.Move(rutaOriginal, nuevaRuta)

Utilizamos la biblioteca IO.Path de .NET para garantizar que la ruta del archivo sea válida y no se pierdan datos durante el proceso de renombrado.

2. Algoritmo de Compresión (ZIP)

Para la reducción de tamaño, implementamos la clase ZipFile, que permite agrupar múltiples archivos en un solo paquete comprimido, facilitando su envío y almacenamiento. 


' Comprimir una carpeta completa a un archivo ZIP
Dim rutaCarpeta As String = "C:\Proyecto\Datos"
Dim rutaZip As String = "C:\Proyecto\Backup.zip"

IO.Compression.ZipFile.CreateFromDirectory(rutaCarpeta, rutaZip)

3. Sistema de Filtrado Inteligente

Para facilitar la selección de archivos, el software implementa un filtro dinámico que permite visualizar únicamente los formatos compatibles con la operación deseada (PDF, ZIP, TXT, etc.). 


' Configuración del filtro para el cuadro de diálogo
Dim openFileDialog As New OpenFileDialog()
openFileDialog.Filter = "Archivos de Texto (*.txt)|*.txt|Documentos PDF (*.pdf)|*.pdf|Todos los archivos (*.*)|*.*"
openFileDialog.FilterIndex = 1

If openFileDialog.ShowDialog() = DialogResult.OK Then
    Dim ruta As String = openFileDialog.FileName
    ' Lógica para procesar el archivo seleccionado
End If

4. Indicador de Progreso (ProgressBar)

Durante las operaciones de compresión pesadas, la aplicación informa al usuario en tiempo real mediante una barra de progreso, mejorando la interacción y el feedback del sistema. 


' Actualización de la interfaz durante un bucle de procesamiento
For i As Integer = 1 To totalArchivos
    ProcesarArchivo(i)
    
    ' Actualizar la barra de progreso en la UI
    ProgressBar1.Value = (i / totalArchivos) * 100
    lblEstado.Text = "Procesando: " & i & " de " & totalArchivos
Next

6. Gestión de Rutas de Salida

El usuario puede elegir dónde desea guardar los archivos procesados. Esta función utiliza el componente FolderBrowserDialog para una navegación segura por las carpetas del sistema. 


' Selección de carpeta de destino para archivos convertidos
Dim selectorCarpeta As New FolderBrowserDialog
selectorCarpeta.Description = "Seleccione la carpeta donde guardar los resultados"

If selectorCarpeta.ShowDialog() = DialogResult.OK Then
    rutaDestinoGlobal = selectorCarpeta.SelectedPath
    lblRutaSeleccionada.Text = "Destino: " & rutaDestinoGlobal
End If

7. Optimización y Limpieza

Para no saturar el almacenamiento del usuario, el software incluye un proceso de limpieza que elimina archivos temporales una vez finalizada la compresión o conversión. 


' Eliminar archivos temporales después del proceso
If IO.File.Exists(rutaTemporal) Then
    Try
        IO.File.Delete(rutaTemporal)
    Catch ex As Exception
        ' Error silencioso si el archivo está en uso
    End Try
End If

8. Integración con Formato PDF

Una de las funcionalidades estrella del cambiador es la capacidad de preparar documentos para su exportación a PDF, asegurando que la estructura del archivo se mantenga íntegra durante la conversión. 


' Lógica para verificar la integridad del PDF antes de la conversión
If IO.Path.GetExtension(rutaArchivo).ToLower() = ".pdf" Then
    ' El software valida que el archivo no esté protegido por contraseña
    Console.WriteLine("Archivo PDF detectado y listo para procesar.")
End If

9. Validación y Seguridad de Datos

Para prevenir errores críticos, el programa valida que los campos no estén vacíos y que las rutas seleccionadas existan realmente en el dispositivo antes de ejecutar cualquier operación. 


' Validación de campos obligatorios
If String.IsNullOrEmpty(txtRuta.Text) Or String.IsNullOrEmpty(txtExtension.Text) Then
    MessageBox.Show("Por favor, rellene todos los campos antes de continuar.", 
                    "Atención", MessageBoxButtons.OK, MessageBoxIcon.Warning)
    Return ' Detiene la ejecución
End If

10. Finalización del Ciclo de Vida

Al cerrar la aplicación, el sistema se asegura de liberar todos los procesos de fondo relacionados con la compresión para no dejar hilos de ejecución huérfanos en Windows. 


Private Sub Form1_FormClosing(sender As Object, e As FormClosingEventArgs) Handles MyBase.FormClosing
    ' Liberar recursos y cerrar flujos de datos abiertos
    Application.ExitThread()
    End
End Sub

11. Eliminación Segura de Archivos

Para proteger los datos del usuario, el software no elimina los archivos de forma permanente de inmediato, sino que utiliza las librerías de Windows para enviarlos a la papelera, permitiendo su recuperación en caso de error. 


' Enviar archivo a la papelera de reciclaje en lugar de borrarlo permanentemente
My.Computer.FileSystem.DeleteFile(rutaArchivo, 
    Microsoft.VisualBasic.FileIO.UIOption.OnlyErrorDialogs, 
    Microsoft.VisualBasic.FileIO.RecycleOption.SendToRecycleBin)

12. Guía de Interacción para el Usuario

La interfaz ha sido diseñada para ser autodescriptiva. Siguiendo estos tres pasos, cualquier usuario puede dominar la herramienta en segundos:

  • Seleccionar: Use el botón de búsqueda para localizar sus archivos.
  • Configurar: Elija el idioma y el formato de destino (ZIP o extensión).
  • Ejecutar: Pulse el botón central para iniciar el proceso automatizado.
